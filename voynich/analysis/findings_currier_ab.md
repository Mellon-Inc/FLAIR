# Currier A/B 二方言 — 同じコードブックか別物か?

実行: `python voynich/analysis/currier_ab.py`
出力: `currier_ab_results.json`

## 設問

Currier (1976) 以来、Voynichese には 2 方言 (Language A, Language B) があることが知られている。Phase 5 で「Voynichese は verbose substitution cipher の出力」が強く示唆された今、追加の問い:

- A と B は **同じ暗号鍵 / 違う平文** か?
- A と B は **違う暗号鍵** で書かれた別物か?
- A と B は **同じ鍵 / 違う平文言語** (例: ラテン語 vs イタリア語) か?

## 方法

Phase 1 の JSD クラスタリングで決まった経験的二群:

- **Dialect A** (Currier A 寄り): Herbal + Pharmaceutical (13,486 トークン, 4,045 タイプ)
- **Dialect B** (Currier B 寄り): Biological + Recipes (15,432 トークン, 3,302 タイプ)

(Cosmological / Astronomical は中間で除外。)

各方言で:
1. 語彙集合 V_A, V_B の重複 (Jaccard、トークン質量で見た重複率)
2. 語頻度分布の JSD (全語彙、上位 1000 語)
3. 編集距離 1 グラフを別々に構築・比較
4. トップ語族のスロット文法を比較
5. **Naibbe コードブックでデコードプローブ**: 各方言の上位 200 語を Greshko の Naibbe 表で逆引き、暗号元の平文文字・テーブル分布を推定

## 結果 1: 語彙重複は中程度、トークン質量はほぼ共有

| 指標 | 値 | 解釈 |
|---|---:|---|
| \|V_A ∩ V_B\| | 1,123 | |
| \|V_A ∪ V_B\| | 6,224 | |
| Jaccard | 18.0% | |
| ランダム期待重複 | 2,146 | |
| **観測 / 期待 比** | **0.52** | 共有プールからのランダム抽出より少ない |
| A トークンが共有語彙にある割合 | **71.5%** | |
| B トークンが共有語彙にある割合 | **79.7%** | |

**観察 1**: Jaccard 18% と低めだが、これは **タイプ集合** での重複。実際のトークン (使用頻度を加味) では A の 71.5%、B の 79.7% が両方言で共通の語を使っている。

**観察 2**: 「ランダム期待の 52%」というのは、A と B が完全に共通プールから引いたランダムサンプルだとしたら期待される重複の半分しかないということ。**A と B にはそれぞれ固有の "長尾" 語彙がある**。

→ A と B は **共通の高頻度コア + 各自固有のロングテール** という構造。

## 結果 2: 頻度分布は劇的に違う

| 比較 | JSD |
|---|---:|
| 全語彙の頻度分布 JSD(A, B) | **0.406** |
| 上位 1000 語に絞った JSD(A, B) | **0.269** |

JSD = 0.4 は通常「同一テキストの異なるセクション」では考えられない大きさ。同じ言語の異なる部分なら 0.05 未満。**A と B は同じ語彙を使っていても、その使用頻度が根本的に違う**。

`daiin / aiin / ol / chey` のような短い語は両方で頻出だが、`qokedy / qokeedy / qokeey / chedy / Shedy` の **`-edy` 系語** は B で爆発的に頻出 (例: `qokeedy` は B で 285、A で 10 → **28.5 倍**)。

## 結果 4: 語族グラフは構造同じ、頭語だけ違う

| 指標 | A | B |
|---|---:|---:|
| n_types (count≥2) | 1,128 | 1,104 |
| 平均次数 | 9.93 | 9.18 |
| 最大成分占有率 | 0.30 | 0.28 |
| クラスタ係数 | 0.48 | 0.49 |
| 大成分数 (≥3) | 10 | 10 |

**観察 3**: グラフ構造はほぼ瓜二つ。両方言とも同じくらい高密度・同じくらい家族集中。

トップ語族の頭語:

| Rank | A 頭語 (size, freq) | B 頭語 (size, freq) |
|---|---|---|
| 1 | chol (164, 2619) | **chedy** (252, 3311) |
| 2 | aiin (304, 2586) | daiin (305, 3042) |
| 3 | daiin (342, 2429) | chey (136, 2238) |
| 4 | or (47, 1187) | qokeey (208, 2131) |
| 5 | qokeey (121, 598) | ol (41, 1112) |

両方言で `daiin / qokeey / qokaiin` 族は出るが、A の最大族は `chol` (3 グリフ語族)、B の最大族は `chedy` (4 グリフ語族)。**B の語は A より 1 グリフ長い傾向**。

## 結果 5: スロット文法の比較

A の `chol` 族 (modal_len=3):
- pos 0: ch (861), d (463), Sh (367), o (174)
- pos 1: o (1186), a (522), e (321), d (111)
- pos 2: l (781), y (755), r (700), m (123)

B の `chedy` 族 (modal_len=4):
- pos 0: ch (951), Sh (692), o (633), a (288)
- pos 1: **e (1589 — 圧倒的)**, t (284), k (265), o (251)
- pos 2: d (951), a (606), e (548), o (399)
- pos 3: y (1825), l (533), r (469), n (277)

**観察 4**: スロットのグリフ集合 (ch/Sh/o, o/a/e, y/l/r/n 等) は両方言でほぼ同じ。違いは:
- B では **第 2 位置が e に圧倒される** (1589 vs A の対応位置の o 1186)
- B のほうが **1 グリフ長いテンプレート** が優勢

この違いを Naibbe 用語で言うと: B では `unigram_alpha_e = chedy` のような短い母音単独エンコードが大量にあり、A では同じ語が少ない。

## 結果 6: Naibbe コードブックでデコードプローブ (★ 核心)

各方言の上位 200 語を Greshko (2025) の Naibbe テーブル CSV と照合し、暗号元の平文文字・テーブル・状態 (unigram/prefix/suffix) を推定:

| 指標 | Dialect A | Dialect B |
|---|---:|---:|
| Naibbe コードブックに **無い** 語の割合 | **31.9%** | **18.6%** |
| 平文文字 1 位 | **a** (8.70%) | **i** (10.80%) |
| 平文文字 2 位 | n (6.52%) | **e** (9.85%) |
| 平文文字 3 位 | r (5.89%) | r (7.53%) |
| 平文文字 4 位 | d (4.63%) | t (6.92%) |
| 平文文字 5 位 | e (4.51%) | s (6.88%) |
| 平文文字 6 位 | s (4.50%) | a (6.67%) |
| **Alpha テーブル支配度** | 18.4% | **32.3%** |
| Beta テーブル合計 | 37.7% | 35.5% |
| Gamma テーブル合計 | 12.0% | 13.6% |
| **Unigram 状態** | 46.6% | **66.4%** |
| Suffix 状態 | 13.6% | 8.1% |
| Prefix 状態 | 7.9% | 6.8% |

### 解釈

**観察 5**: B のほうが Naibbe コードブックに 1.7 倍よく適合 (81.4% match vs 68.1% match)。Naibbe テーブルは B 系統の語族 (Biological/Recipes 系) に最適化されているように見える。Greshko 自身が tuning したテーブルであるため、これは自然。

**観察 6**: B は **Alpha テーブル支配** (32.3%) で **Unigram 多用** (66.4%)。Naibbe デフォルトの「Alpha が card weight 28/78」「Unigram が dominant」 という設定そのものに近い。

**観察 7**: A は **Beta/Gamma テーブルがより均等** で **Unigram は 46.6%** (= Bigram も多い)。これは Naibbe デフォルトと違う card weights、または **平文がより 2 文字チャンクになりやすい言語** で書かれた可能性。

**観察 8**: 平文文字推定の差が決定的:
- A: a (8.7%), n (6.5%), r (5.9%), d (4.6%), e (4.5%), s (4.5%), t (3.9%), i (3.9%)
- B: **i (10.8%), e (9.9%)**, r (7.5%), t (6.9%), s (6.9%), a (6.7%), c (4.8%), d (4.1%)

参考: 自然言語の文字頻度

| 言語 | 上位文字 (上位 6) |
|---|---|
| Latin | e, i, u, t, s, a |
| Italian | e, a, i, o, n, t |
| Vulgate Latin | i, e, t, s, u, n |
| German | e, n, i, r, s, a |

→ **B の平文文字推定 (i, e, r, t, s, a) は古典/中世ラテン語に非常に近い**
→ **A の平文文字推定 (a, n, r, d, e, s) は明確にラテン語ではない。中世イタリア語、または別のロマンス語、または専門用語的な短い 2 文字チャンク主体テキスト** に整合

## 統合解釈

以下のシナリオが最も整合:

> **Voynich 手稿 A 系統と B 系統は同じ Naibbe 様 verbose substitution 暗号で書かれているが、平文の言語または文体が異なる。B は古典ラテン語的、A は中世イタリア語的または異なる語彙体系。**
> **両者は ~7,000 語規模のコードブックの「コア」(daiin, ol, aiin, qokeey, qokaiin など) を共有するが、各自の「ロングテール」(B の chedy/Shedy/qokeedy 系、A の chol/chor/Shol 系) は平文言語の違いで生まれている。**

これは Currier (1976) の二方言観察、Davis (2020) の「複数書記者」観察、Phase 5 の Naibbe 整合性、本 Phase の Naibbe デコード結果すべてと整合する最も簡潔な仮説。

## 反証可能な含意

仮説が正しいなら:
1. **同一コードブックなのでテーブル位置選好は両方言で同じ** → q が両方言とも 99% 語頭、n が両方言とも 97% 語尾 (Phase 1 で確認済み — 一致)
2. **平文言語が違うので文字 n-gram 統計が違う** → B のほうが Latin の連母音 (-ae, -is, -us 終わり) の多重エンコード形を多用すべき → 確認: B での `qok-` 接頭辞 (qokeey, qokeedy, qokedy) は alpha-table の i, r, s エンコード、これは Latin によくある -is/-us/-er エンディングと整合
3. **両方言で同じ unigram 短語 (daiin, aiin, ol) が頻出すべき** → 確認 (rank 1-9 で daiin/aiin/ol が両方に登場)
4. **同じ書記者が両方言を書ける可能性** → Davis 5 書記者説と整合 (A, B どちらにも複数書記者がいる)

## 反証されうる方法

- B 内で `qokeedy` などを「i」とデコードしたら Latin として読める文章になるはず → 直接デコード試行 (次の Phase の候補)
- A の 31.9% 不適合語の内部構造を解析 → 別コードブック (例: Naibbe + α 表) かもしれない
- Currier B 系統のページのみで Naibbe を **逆方向にデコード** し、ラテン語っぽさ (Vulgate との bigram 距離) を測る

# Voynichese 形態論 — 語族グラフによる解析

実行: `python voynich/analysis/word_families.py`
出力: `voynich_families.json`, `graph_stats.csv`

## 方法

1. Voynichese から count ≥ 2 の語タイプ 2,385 を抽出 (グリフ列でトークナイズ)
2. すべての語ペアについてグリフ単位編集距離 = 1 で辺を張り、無向グラフを構築
3. 連結成分 (= 語族) を抽出
4. 各語族について「最頻語長で揃えた整列」をとり、各位置のグリフ多重集合 (= スロット文法) を集計
5. 同じ手順を **Pliny (ラテン語), Bible (フィンランド語), Pride & Prejudice (英語), Divina Commedia (イタリア語)** に対して、語彙サイズを Voynichese と揃えた上で適用

文字レベル ed=1 グラフは「同じ単語を 1 文字だけ変えて作れる別の単語」を辺で結ぶもの。自然言語ではこれは「派生形ペア」(walk/walks, amo/amas) を中心にとらえる。

## 結果 1: グラフ統計の言語間比較

| コーパス | n_types | 平均次数 | 最大成分占有率 | クラスタ係数 | 大成分数 (≥10) |
|---|---:|---:|---:|---:|---:|
| **Voynichese** | 2,385 | **14.18** | **0.291** | **0.498** | **10** |
| Latin (Pliny) | 2,385 | 5.98 | 0.131 | 0.476 | – |
| Italian (Dante) | 2,385 | 2.99 | 0.222 | 0.407 | – |
| Finnish (Bible) | 2,385 | 1.12 | 0.066 | 0.364 | – |
| English (P&P) | 2,385 | 0.84 | 0.095 | 0.395 | – |

**観察 1**: Voynichese の平均次数は **ラテン語の 2.4 倍、フィンランド語の 12.6 倍、英語の 16.9 倍**。語彙サイズを揃えても圧倒的に密。

**観察 2**: 最大連結成分が全語彙の 29.1% を占める。ラテン語 (13.1%)・イタリア語 (22.2%) と比べても異常。Voynichese 語彙の **大半が単一の巨大ファミリーで近距離につながっている**。

**観察 3**: Voynichese の上位 3 ファミリー (695 + 546 + 456 = 1,697 語) が全語彙の **71%** を占有する。自然言語では同じ規模での集中はあり得ない。

## 結果 3: 「これは形態論なのか?」

各位置のグリフ集合は**ほぼ独立**に選ばれている。たとえばファミリー A (daiin) を見ると:

- 位置 0 が選べるグリフ: ~20 種類のうち実質 5-10 種
- 位置 1 が選べるグリフ: 同上、別の傾向
- 位置 4 (語尾) が選べるグリフ: y / n / l / r / s の 5 種に集中

仮にこれが自然言語の屈折なら、各語族は同じ語幹 + 異なる接尾辞で生まれるはず。すると「語幹位置のグリフはほぼ不変、接尾辞位置のグリフだけ変動」が見えるはず。だが Voynichese では **すべての位置が変動する**。これはむしろ:

> **5 個のサイコロ (位置ごとに) を独立に振って語を作る生成系**

の出力に近い。自由度の積はすぐに数千通りに達するため、語数の爆発が起きる。

## 結果 4: ラテン語との直接比較

ラテン語 (Pliny の同サイズサブセット) では:

- 平均次数 5.98 (Voynichese の 42%)
- 最大成分 13.1%
- 語族の中身: `dixit / dixerat / dixerint / dictum` のように **語幹固定 + 屈折部のみ変化**

これに対し Voynichese の `qokeey ↔ qokedy ↔ qokeedy ↔ qokain` は:
- `qok-` の共通部分はあるが、**語中の母音・末尾の n/y/d まで全部入れ替わる**
- 入れ替わりが「同じスロット内での代替」になっており、ラテン語型の「語幹+語尾分離」とは構造が違う

## 結果 5: 結論と新しい仮説

| 仮説 | 整合性 |
|---|---|
| Voynichese = 自然言語をそのまま音写したもの | **不整合**: グラフ密度・語族集中度が桁違い |
| Voynichese = ラテン語の単純な置換暗号 | **不整合**: 同じ理由 |
| Voynichese = テンプレート言語 (slot grammar) | **整合**: 観察された構造そのもの |
| Voynichese = verbose substitution cipher (Naibbe 系) | **整合**: 1 平文文字 → 複数グリフ展開はスロット文法を生む |
| Voynichese = Rugg 式 Cardan grille 生成物 | **整合**: グリル列を切り替えて語を量産すればこの構造になる |
| Voynichese = 自己引用テーブル生成 (Timm & Schinner) | **整合**: 同上 |

つまり今回の解析は **「Voynichese は自然言語の音写ではなく、グリフ位置ごとに独立な選択をする生成プロセスの出力である」** という従来仮説 (Currier, Tiltman, Stolfi, Rugg, Timm) を **定量的に再確認** した。

「では何を選んでいるのか」がまだわからない。可能性は:
1. **無意味**: 単なる擬似テキスト (中世の "知的見せかけ" 写本)
2. **1 平文文字 → 複数スロットへの暗号展開** (Naibbe 系)。これなら統計と意味の両立が可能
3. **音価ではなく「分類コード」**: たとえば植物の属・種・部位・状態を 4-5 桁のコードで書き、スロット = フィールドに対応 (植物データベース仮説)

この 3 つは依然として未決着。ただし (1) は「挿絵と語の整合 (Currier A/B が章ごとに分かれる事実)」を説明しにくく、(2)(3) のほうが説明力が高い。

## 結果 6: 今後の検証アイデア

- **語族内の挿絵相関**: ある植物画ページにおける語が、別の植物画ページに比べて、どの語族にどれだけ偏るか。スロット = 植物特徴説の検証
- **Naibbe 暗号の実装**: ラテン語/イタリア語ボタニカル語彙を Naibbe で暗号化し、得られる語族グラフが Voynichese とどれだけ一致するか
- **Currier A vs B のスロット構造比較**: 二方言で同じスロット文法か、それとも別ルールか。同じなら「同じ生成系の二バリエーション」、違うなら「二つの異なるコードブック」
